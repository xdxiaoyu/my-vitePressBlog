## 🔍 如何测试图片是否加载

### 方法 1：浏览器直接访问

启动开发服务器后，在浏览器访问：

- http://localhost:5173/bg.jpg - ✅ 应该直接显示图片
- http://localhost:5173/bg1.png - ✅ 应该直接显示图片
- http://localhost:5173/favicon.ico - ✅ 应该直接显示图标

如果上述 URL 能正常访问，说明 public 目录配置正确。

### 方法 2：使用浏览器开发者工具

1. 打开浏览器开发者工具（F12）
2. 切换到 **Network** 标签
3. 筛选 **Img** 类型
4. 刷新页面
5. 查看图片请求状态：
   - ✅ 绿色/200：加载成功
   - ❌ 红色/404：文件不存在
   - ⚠️ 黄色/待定：正在加载或超时

### 方法 3：检查控制台错误

在开发者工具的 **Console** 标签中，查看是否有：
- 404 错误（文件不存在）
- CORS 错误（跨域问题）
- 网络错误（连接问题）

## ⚠️ 注意事项

### 1. 图片路径规则

在 VitePress 中引用 public 目录的文件：

```markdown
✅ 正确：![图片](/bg.jpg)
❌ 错误：![图片](./bg.jpg)
❌ 错误：![图片](../public/bg.jpg)
❌ 错误：![图片](public/bg.jpg)
```

**关键**：路径必须以 `/` 开头

### 2. 图片大小优化

`bg3.jpg` 有 4.6MB，建议压缩：

```bash
# 使用 ImageMagick（需要先安装）
convert bg3.jpg -quality 85 -resize 1920x bg3-optimized.jpg

# 或使用在线工具：
# - TinyPNG: https://tinypng.com
# - Squoosh: https://squoosh.app
```

### 3. 浏览器缓存

如果更新了图片但浏览器仍显示旧图片：

1. 硬刷新：`Ctrl+F5` (Win) 或 `Cmd+Shift+R` (Mac)
2. 清除缓存：浏览器设置 → 清除浏览数据
3. 重命名文件（改变文件名会强制浏览器重新加载）
